Running Metropolis algorithm

We import the necessary libraries from our repository (stored in the file “N-cylinder/MetropolisAlgorithm.R”).

Now we set the necessary parameters. If we forget to set the value of some parameter, it will take the default values (set in “MetropolisAlgorithm.R” file)

params = list(
  seed = 42, # 
  steps = 2, # simulation steps
  n = 100, # number of cells
  L=3, # numbers of layers
  RadiusA = 5/(2*pi), # apical radius
  Ratio = 2.5, # ratio between apical and basal radius 
  cyl_length = 20, # cylinder length
  gamma_ad = 0.15, #  contractility
  lambda_ad = 0.04, # adhesion 
  # elastic K = 1 cause adimentional
  beta = 100, #
  s0=1 # 
  )

The function that performs Metropolis Algorithm is called “metropolisad” (the ad suffix refers to the adimensionality of the algorithm). Now we do a call of the function with the parameters desired.

result_alg <- do.call(metropolisad,params)

Now we have the desired results stored in the variable “result_alg”. Let’s deep into this variable, which has 3 elements: the history of points locations in every step, the history of the energy at every step, and the value of nu squared, which is a metric over the topology of the surface.

We see that hist_energy contains simply 2 columns, one indicating the iteration (or step or Frame) and other indicating the energy after performing that number of iterations of Metropolis. In our case, if we execute the algorithm for 250 steps we will have 250 rows.

Analysis of the results

Now let’s analyze the results we have got. First we import our library that contains the necessary code and functions.

rad_coef = params$Ratio
steps = params$steps
Radius = params$RadiusA
Radius2 <- rad_coef*Radius
cyl_thickness <- Radius2-Radius
cyl_length = params$cyl_length
cyl_width = Radius*(2*pi)
rec <- list()
rad <- list()
L = params$L
n = params$n
A0 <- ((Radius2+Radius)*pi*cyl_length)/n
xmin <- 0
ymin <- 0
xmax <- cyl_width
ymax <- cyl_length
for(k in 1:L){
  rad[[k]]<- Radius+(k-1)*(cyl_thickness/(L-1)) #the radius of the layer k
  rec[[k]]<-c(xmin,xmin+3*(2*pi*rad[[k]]),ymin,ymax)
}
lamad = params$lambda_ad
gamad = params$gamma_ad

The function energy_analisis_1sim takes as argument the points in each iteration and returns the energy shelled by type in each iteration (an extension of the energy history). The columns represents the elastic, contractile and tensile energy, together with the sum of all (the total energy). The function plot_energyss takes the output of this function and makes a plot
